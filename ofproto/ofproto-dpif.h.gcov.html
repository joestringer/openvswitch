     161                 :            :  * Recirculation
     162                 :            :  * =============
     163                 :            :  *
     164                 :            :  * Recirculation is a technique to allow a frame to re-enter the packet
     165                 :            :  * processing path for one or multiple times to achieve more flexible packet
     166                 :            :  * processing in the data path. MPLS handling and selecting bond slave port
     167                 :            :  * of a bond ports.
     168                 :            :  *
     169                 :            :  * Data path and user space interface
     170                 :            :  * -----------------------------------
     171                 :            :  *
     172                 :            :  * Two new fields, recirc_id and dp_hash, are added to the current flow data
     173                 :            :  * structure. They are both of type uint32_t. In addition, a new action,
     174                 :            :  * RECIRC, are added.
     175                 :            :  *
     176                 :            :  * The value recirc_id is used to distinguish a packet from multiple
     177                 :            :  * iterations of recirculation. A packet initially received is considered of
     178                 :            :  * having recirc_id of 0. Recirc_id is managed by the user space, opaque to
     179                 :            :  * the data path.
     180                 :            :  *
     181                 :            :  * On the other hand, dp_hash can only be computed by the data path, opaque to
     182                 :            :  * the user space. In fact, user space may not able to recompute the hash
     183                 :            :  * value. The dp_hash value should be wildcarded when for a newly received
     184                 :            :  * packet. RECIRC action specifies whether the hash is computed. If computed,
     185                 :            :  * how many fields to be included in the hash computation. The computed hash
     186                 :            :  * value is stored into the dp_hash field prior to recirculation.
     187                 :            :  *
     188                 :            :  * The RECIRC action computes and set the dp_hash field, set the recirc_id
     189                 :            :  * field and then reprocess the packet as if it was received on the same input
     190                 :            :  * port. RECIRC action works like a function call; actions listed behind the
     191                 :            :  * RECIRC action will be executed after its execution.  RECIRC action can be
     192                 :            :  * nested, data path implementation limits the number of recirculation executed
     193                 :            :  * to prevent unreasonable nesting depth or infinite loop.
